# TheMauler — UI / UX Audit (2026-06-11)

Audience: a follow-up AI/engineer who will implement the changes, plus the operator.
Scope: layout, density, clipping/overflow, interaction affordances (tooltips, spell-check,
right-click), theming consistency, and overall human readability — making the UI faster and
easier for *you* to read and operate during long sessions. Findings-only where noted;
nothing here is committed.

Operator notes that drove this audit:
- The **card/"bubble" style in Logs/Ops wastes space** — wants denser plain text/tables to
  scan more at once.
- **Elements get cut off or hidden under others.**
- Missing **hover tooltips**, **spell-check / right-click autocorrect**, and general
  click-level polish.

Stack: React 19 + TS + Vite, custom CSS (no component lib). Runs in **WebView2** (Wails on
Windows) — relevant because WebView2 has native spellcheck + right-click suggestion support
that the app isn't using.

Severity: **U0** = broken/unusable, **U1** = clear daily friction, **U2** = polish,
**U3** = nice-to-have.

## ✅ DONE (2026-06-11, partial) — U0 — Light theme is defined but broken: panels hardcode dark hex instead of tokens

**What landed.** Added semantic surface tokens to `index.css` for both themes — `--c-app`,
`--c-pane`, `--c-panel`, `--c-card`, `--c-card-hover`, `--c-list`, `--c-terminal` — and swept
the hardcoded surface darks (`#151515/#181818/#202020/#242424/#141414/#111820/#111`) across
`App.css` and all `components/*.css` to use them. Major surfaces (workspace, panes, tabs,
log cards, agent panel, terminal) now flip with the theme. Frontend builds clean.
**Remaining (U2 follow-up):** status/accent hexes (`#f87171`, `#fbbf24`, `#4ade80`, `#60a5fa`,
`#fff`, etc.) are not yet tokenized — they're mid-tones that read on both themes, so lower
priority, but tokenizing them would complete light-mode polish.

### Original finding

**Files:** `index.css` defines a full `[data-theme="light"]` palette and `--bg0..--bg3`
tokens, but component CSS hardcodes dark values all over:
`#151515`, `#181818`, `#202020`, `#242424`, `#141414`, `#111`, `#111820`, `#fff`
(e.g. `App.css:7,118,136,206,216`, `LogsPage.css:5,17,64,91,108,292`, `AgentPanel.css`
`#202020`/`#181818`, many others — `grep -rhoE "#[0-9a-fA-F]{3,6}" frontend/src/components/*.css`).

**Effect.** Switching to light theme leaves every major surface (workspace, panes, tabs,
log cards, agent panel) dark — unreadable. The theme toggle looks supported but isn't.

**Fix.** Replace hardcoded surface/border/text hex with the existing tokens
(`var(--bg0/1/2/3)`, `var(--border)`, `var(--text*)`). Add the two or three missing tokens
the panels actually need (e.g. `--surface-raised`, `--surface-sunken`) so both themes flip
cleanly. This is also the precondition for the operator's "more basic text" request — once
surfaces are tokenized, a "compact/plain" density variant is a token swap, not a rewrite.

## ✅ DONE (2026-06-11, Logs) — U1 — Logs & Ops are card/"bubble" stacks; should offer a dense table/plain-text mode

(`LogsPage.tsx`), **defaulting to Compact** and persisted in `localStorage`. Compact mode

**Follow-up fixes (2026-06-11):** corrected a grid-column bug in the compact summary rows
(4 columns declared for 3 cells, which crushed the message column and made long kinds like
`tool_protocol_request` overlap — e.g. "TOOL_PROTOCOChatQUEST"); now `150px / 1fr / 60px`
with ellipsis on kind+message. Also made the **left run-list** cards compact in Compact mode
(tighter padding, single-line prompt) so they're no longer "massive bubbles."
**Remaining:** apply the same compact treatment to the **Ops ACTION LOOP** (`LiveOpsPage.tsx`)
— same pattern, not yet done.

## ✅ DONE (2026-06-11) — U1 (bug) — Markdown file: can't return from Preview to Edit

**Problem.** In the file viewer, the markdown Preview was a single toggle button; once in
preview the only other controls (Format/Diff) were hidden and returning to edit was
unreliable/unclear. **Fix:** replaced it with an explicit, always-visible **Edit | Preview**
segmented control (`FileViewer.tsx` + `.fv-seg` in `FileViewer.css`), so both states are
always one click away. Monaco stays mounted and re-lays-out on exit as before.

### Original finding

**Files:** `LogsPage.css:281-318` (`.logs-event`, `.logs-tool` = rounded `#202020`
`<details>` cards, `border-radius:7px`, `gap:7px`, padded summaries); `LiveOpsPage.tsx`
ACTION LOOP renders rows with pill "outcome" badges; `LogsPage.tsx` timeline/tools.

**Problem (operator's main gripe).** Each event/tool is a bordered, rounded, padded card
with 7px gaps. On a 1080p screen you see ~12–15 rows; the rest scrolls. For triaging a run
with 20–54 tool calls (see the real runs in the screenshots) you can't see volume or scan
patterns. The "bubbles" add chrome, not information.

**Fix — add a compact table view (toggle, default for power users).**
- Render the ACTION LOOP / timeline / tools as a real table or a monospace grid:
  `time · type · action · observation · outcome`, one row each, `font: var(--font-mono)
  12px`, `line-height: 1.35`, 2–4px vertical padding, single hairline row separators (no
  per-row border/radius/background). This roughly **doubles rows-per-screen** and lets the
  eye scan a column (e.g. all `outcome` values) instantly.
- Keep `<details>` expansion for the full payload, but on the row, not a card.
- Replace pill badges with a short uppercase text token in a fixed-width column
  (`DONE` / `ERR` / `STOP`), color via text color only — pills cost ~22px height each.
- Put the toggle in the Logs/Ops header: `[ Cards | Compact ]`. Persist the choice.

This directly answers "I'd rather more basic text to see more issues."

## U1 — Duplicate top-level navigation (titlebar vs center tabs)

**Files:** titlebar nav `App.tsx:691-695` (Chat/Ops/Logs/Memory/Benchmarks) **and** the
center-pane tab strip `App.tsx:745-754` render the *same* five destinations. Visible as two
identical rows in every screenshot.

**Problem.** Two controls for one action; unclear which is canonical; wastes a 40px row and
adds cognitive load. The titlebar also mixes view-switches (Chat/Ops/…) with panel toggles
(Explorer/Agent) and actions (Doctor/Terminal/Settings) in one undifferentiated run of
buttons.

**Fix.** Pick one home for view switching — recommend the **center tab strip** (it's next to
the content it controls) — and remove Chat/Ops/Logs/Memory/Benchmarks from the titlebar.
Reserve the titlebar for: session controls (left), and global actions Doctor/Terminal/
Settings + panel toggles (right), visually grouped with the existing `.titlebar-sep`.

## U1 — Agent panel: last card clipped under the pinned footer, no scroll cue

**Files:** `AgentPanel.css:127-134` (`.agent-tab-body` is the `flex:1` scroll area) and
`:800-806` (`.agent-actions` pinned footer, `flex-shrink:0`). The panel crams Status →
Mode Override (7 buttons) → Behaviour (3 toggle rows) → Access Preset (3 cards) → Toolset →
6 action buttons into one column.

**Problem.** When content exceeds the panel height, the last scrollable card (e.g. the
"Unrestricted — Full enabled tools, no prompts" preset in screenshot 1) sits half-hidden
behind the footer with **no fade, shadow, or scrollbar cue** that more exists. It reads as
"cut off," exactly as reported.

**Fix.**
- Add a scroll affordance: a top/bottom inset shadow or gradient fade on `.agent-tab-body`
  that appears when scrollable (`mask-image` or a sticky `::after`), so the clip reads as
  "scroll for more," not "broken."
- Reduce vertical pressure: collapse Mode Override into a single dropdown or a 2-row chip
  group; make Behaviour toggles a tighter list; consider an accordion so only one section is
  open at a time.
- Give `.agent-actions` a top border + solid background so the boundary between scroll
  content and pinned controls is explicit.

## ✅ DONE (2026-06-11, core inputs) — U1 — Spell-check / right-click autocorrect missing on prose inputs

**What landed.** Added `spellCheck lang="en"` (+ `autoCapitalize="sentences"`) to the prose
inputs: the chat composer (`ChatPane.tsx`) and the Memory note editor (`MemoryPage.tsx`) —
WebView2 now shows the red squiggle and right-click suggestions there. Explicitly set
`spellCheck={false}` + `autoCorrect/autoCapitalize="off"` on the Target IP/URL and VPN
fields (`FileTree.tsx`) so addresses don't get squiggled; the terminal input already had it
off. Frontend builds clean. **Remaining (optional):** apply the same prose/code split to the
agent memory editor and benchmark scenario prompt fields; a shared `<ProseInput>/<CodeInput>`
helper would make the distinction enforceable.

### Original finding

**Files:** only `TerminalPane.tsx:194` sets `spellCheck` (correctly **off** for the
terminal). The chat composer `ChatPane.tsx:395-407` has **no** `spellCheck`, and neither do
the Memory note editor (`MemoryPage.tsx`), agent memory editor, or benchmark scenario
fields. WebView2 supports native red-squiggle spellcheck and right-click "suggestions" when
`spellcheck="true"`.

**Fix.**
- Add `spellCheck lang="en"` (and `autoCapitalize="sentences"`) to **prose** inputs: the
  chat textarea, memory title/body, scenario system/user prompts. WebView2 then gives the
  red squiggle + right-click correction the operator wants, for free.
- Keep `spellCheck={false}` on **code/command/path** fields: terminal, shell command box,
  file paths, model IDs, JSON/schema editors, target IP/VPN. (Add it explicitly so it's not
  left to the default.)
- A short helper component (`<ProseInput>` / `<CodeInput>`) would make the prose-vs-code
  distinction enforceable instead of per-field.

## U1 — Truncated values have no tooltip; can't see the full string

**Files:** ellipsis truncation without `title=` in several places — model id in the Ops
"Live Facts" / status (`Model: Qwen3.6-27B-Uncensored-HauhauCS-A…`), agent root / paths,
preset descriptions, `.logs-metric strong` (`LogsPage.css:239-247` ellipsis), `.center-tabs
button` (`App.css:230-232`), `.logs-time` (`App`/`LogsPage`).

**Problem.** Anything clipped with `text-overflow:ellipsis` is unrecoverable by hover — the
operator can't read the full model name, path, or preset blurb.

**Fix.** Add `title={fullValue}` to every element that can ellipsize (model id, paths, run
titles, preset descriptions, KPI values, tab labels). Cheap, high-value. For AI/agent
readability also expose the full value as an `aria-label` (see the AI section below).

## U2 — Pervasive hardcoded radii/borders; inconsistent control sizing

**Files:** radius is `6px` (`--radius`), `7px`, and `8px` in different files
(`App.css`, `AgentPanel.css`, `LogsPage.css`); button heights vary 26/28/30/34px;
borders sometimes `var(--border)`, sometimes hex.

**Fix.** Standardize on the tokens: one control height scale (e.g. 28px default, 30px
primary), `--radius` for inputs/buttons, `--radius-lg` for cards. A 30-minute sweep removes
most of the visual noise and makes the "compact" density mode trivial.

## U2 — `--text-dim` contrast is low on dark surfaces

**Files:** `--text-dim: #858585` on `#151515`/`#181818` surfaces (used for labels, meta,
timestamps, run subtitles throughout). Ratio ≈ 3.3:1 — under the 4.5:1 WCAG AA text
threshold; the many 10–11px uppercase labels are the worst case.

**Fix.** Lighten dim text to ~`#9d9d9d`+ on dark, or reserve `#858585` for ≥12px non-essential
text only. Bump the 10px uppercase section labels to 11px/600.

**Files:** fixed bottom status bar (`StatusBar.css`) + full-height pages

## U3 — Smaller polish

- **Resize/collapse is hidden:** the only hint that panes collapse is a `title=` on the
  resize handle (`App.tsx:738,807,840`). Add a visible grab affordance (dots) and a small
  chevron toggle so users discover collapse without hovering the 4px handle.
- **Right-click menus only in FileTree** (`FileTree.tsx`). Chat messages and log rows have
  no copy-on-right-click; add a minimal context menu (Copy text / Copy as JSON) — also helps
  the operator pull loot/output quickly during a box.
- **Run-context fields** (Target IP/URL, VPN) are unlabeled text inputs with placeholder-only
  labels (`App` left rail) — placeholders vanish on focus; add persistent labels.
- **Hover states** exist for buttons but custom controls (preset cards, KPI tiles, toggle
  rows) lack a hover cue; add a subtle `background`/`border` hover so they read as clickable.

## Make it easier on the eyes (human readability)

The UI is dense and runs long sessions — these reduce the effort of actually reading it:

- **Text, not just color.** Run state and tool outcomes are shown as colored pills
  (`.logs-status.ok/.warn/.bad`, outcome badges) where the meaning lives in the hue. At a
  glance — especially while scanning fast or tired — color alone is easy to misread, and it
  fails entirely for colorblind users (~8% of men). Always pair the color with the literal
  word (`DONE` / `ERROR` / `STOPPED`). The compact-table view in U1 does this by default.
- **Bigger hit/scan targets for the important stuff.** The 10px uppercase section labels and
  11px meta text are hard to read at a glance; bump the essential ones (run state, phase,
  outcomes) to 12px+ and normal case so they register without squinting.
- **Don't hide meaning in hover-only places.** Truncated model ids/paths, the collapse
  handles, and preset blurbs only reveal themselves on hover (or not at all) — see the
  tooltip and resize-affordance items above. Persistent labels beat hover for anything you
  read often.
- **Quieter chrome, louder content.** Right now borders, pills, and rounded cards compete
  with the actual log text for attention. Flattening them (U1/U2) makes the words you care
  about — the command, the output, the error — the most visible thing on screen.
